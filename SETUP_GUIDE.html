🔥 Firebase Setup Guide

Follow these steps exactly. Takes ~15 minutes. Completely free.

1

Create a Firebase Project

Go to console.firebase.google.com and sign in with your Google account.

Click "Add project" → Name it "LIC-Reminder" → Disable Google Analytics (not needed) → Click "Create project".

✓ Wait for it to say "Your new project is ready" then click Continue.
2

Add a Web App to get your Config

Inside your project, click the Web icon (</>) to add a web app.

Give it a nickname like "LIC App" → Click "Register app".

You will see a config block like this — copy the entire object:

const firebaseConfig = { apiKey: "AIzaSy...", authDomain: "lic-reminder.firebaseapp.com", projectId: "lic-reminder", storageBucket: "lic-reminder.appspot.com", messagingSenderId: "123456789", appId: "1:123456789:web:abc123" };
⚠️ Paste this config into BOTH agent.html and admin.html where it says PASTE_YOUR_apiKey_HERE etc.
3

Enable Authentication

In the left sidebar click Authentication → Get started.

Under "Sign-in method" tab, enable these two:

  • Email/Password — click it → toggle Enable → Save
  • Phone — click it → toggle Enable → Save
✓ Both providers should now show as "Enabled".
4

Create Firestore Database

In the left sidebar click Firestore Database → Create database.

Select "Start in production mode" → Choose region asia-south1 (Mumbai) → Click Enable.

⚠️ Now you must set up Security Rules or the app won't work.

After database is created, click the "Rules" tab and replace everything with:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Agents can read/write their own profile match /agents/{agentId} { allow read: if request.auth != null && request.auth.uid == agentId; allow write: if request.auth != null && request.auth.uid == agentId; // Agents can read/write their own clients match /clients/{clientId} { allow read, write: if request.auth != null && request.auth.uid == agentId; } } // Admin can read/write all agents match /agents/{agentId} { allow read, write: if request.auth != null && request.auth.uid == "PASTE_ADMIN_UID_HERE"; } } }

Click Publish. (We'll fill PASTE_ADMIN_UID_HERE in Step 6.)

5

Create Your Admin Account

In Firebase console → Authentication → Users tab → Click "Add user".

Enter your email and a strong password → Click Add user.

✓ This is your personal admin login. Keep this email & password safe.
6

Get Your Admin UID

Open admin.html in your browser. Login with the email/password you just created.

A popup will appear showing your UID — it looks like: abc123xyz789...

Copy that UID and paste it in two places:

  • In admin.html where it says PASTE_YOUR_ADMIN_UID_HERE
  • In the Firestore Rules where it says PASTE_ADMIN_UID_HERE
⚠️ After pasting your UID in admin.html, reload the page. Now only YOU can access the admin panel.
7

Add Your First Agent

In admin.html, tap "+ Add Agent" → fill their name, email, expiry date → Save.

Then the agent needs to:

  • Open agent.html in their browser
  • First time: Go to Firebase Console → Authentication → Add user with their email + set a password → tell them the password
  • They login on agent.html and start adding their clients
💡 Tip: You can also use Firebase's "Send password reset email" button to let agents set their own password without you seeing it.
8

Agent First Login — Linking Step

When an agent logs in for the first time, their Firebase UID is created. The admin panel stores agents by UID, so there's one small step:

  • Agent logs in → their UID is generated by Firebase Auth
  • You need to go to Firestore Database in Firebase console
  • Find the agent's document (created when you added them) and note their UID from Firebase Auth
  • Either rename the document to their UID, or use the admin panel's Edit → the UID is shown there
⚠️ This is only needed once per agent. After linking, everything works automatically.
✓ Easier way: When agent logs in, their data is stored under their UID automatically. Just make sure the document ID in Firestore matches their Firebase Auth UID.
9

Disabling / Renewing Agent Access

In admin.html:

  • Disable access — tap 🔴 Disable Access → agent is immediately locked out next time they open the app
  • Re-enable — tap ✅ Enable Access
  • Change expiry — tap ✏️ Edit → change date → Save
💡 Expiry is checked every time the agent opens the app. Even mid-session, if you disable them they'll see the "Access Revoked" screen on next page load.
10

How to Share the App

The simplest free option — use GitHub Pages (free hosting):

  • Create a free account at github.com
  • Create a new repository named lic-app
  • Upload agent.html and admin.html
  • Go to Settings → Pages → Source: main branch → Save
  • Your app will be live at: yourusername.github.io/lic-app/agent.html
✓ Share the agent.html URL with agents. Bookmark admin.html for yourself. Both work on mobile browsers perfectly.

🎉 You're all set!
